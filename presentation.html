Introduction

What is Newsly?

🎯

Purpose

Newsly is a modern, responsive news web application that fetches real-time news headlines from the internet and displays them in a beautiful UI with user authentication.

👤

User Features

Users can register/login, browse breaking news by category, search for any topic, and save articles to read later — all in one place.

🌐

Real-time Data

News articles are fetched live from NewsAPI.org — a third-party news aggregator covering hundreds of sources worldwide.

🎓

Project Type

3rd Year B.E./B.Tech Computer Science project demonstrating full-stack MERN development with REST APIs, JWT security, and database persistence.

Backend — server.js

The Entry Point — server.js

import express from 'express'; import dotenv from 'dotenv'; import cors from 'cors'; import connectDB from './config/db.js'; dotenv.config(); connectDB(); const app = express(); const PORT = process.env.PORT || 5000; app.use(cors({ origin: ['http://localhost:5173'] })); app.use(express.json()); app.use('/api/auth', authRoutes); app.use('/api/news', newsRoutes); app.use('/api/saved', savedRoutes); app.listen(PORT, () => console.log(`Server running`));
  • 1
    dotenv.config() — Loads secret keys from .env file (DB URL, JWT secret, NewsAPI key)
  • 2
    connectDB() — Connects to MongoDB Atlas before anything else runs
  • 3
    CORS — Allows frontend (port 5173) to talk to backend (port 5000). Without this, browser blocks requests.
  • 4
    express.json() — Parses incoming JSON request bodies so we can read req.body
  • 5
    3 Route modules — Auth, News, Saved. All prefixed with /api/
  • 6
    app.listen(5000) — Starts the HTTP server on port 5000

Backend — Authentication

How User Auth Works

User Model (MongoDB Schema)

// models/User.js const userSchema = Schema({ name: { type: String, required: true }, email: { type: String, unique: true }, password: { type: String, required: true }, isAdmin: { type: Boolean, default: false } }); // Auto-hash password before saving userSchema.pre('save', async function() { const salt = await bcrypt.genSalt(10); this.password = await bcrypt.hash(this.password, salt); });

JWT Flow (Login → Access)

1️⃣ POST /api/auth/register — User sends name + email + password
↓
2️⃣ bcrypt hashes the password & saves User to MongoDB
↓
3️⃣ Server returns JWT Token (valid 30 days)
↓
4️⃣ Frontend stores token in localStorage
↓
5️⃣ Every protected request sends: Authorization: Bearer <token>
↓
6️⃣ authMiddleware.js verifies token → grants access

Complete Data Flow

Trace a request from the browser all the way to NewsAPI.org and back:

1️⃣   User visits HomePage in browser
↓
2️⃣   React component runs: axios.get('/api/news/headlines') inside useEffect
↓
3️⃣   Express server receives GET request at /api/news/headlines
↓
4️⃣   newsController.js calls NewsAPI: axios.get('https://newsapi.org/v2/top-headlines?apiKey=...')
↓
5️⃣   NewsAPI returns JSON array of articles to backend
↓
6️⃣   Backend sends res.json(articles) to frontend
↓
7️⃣   React sets state: setArticles(data) → UI re-renders with news cards

Concepts

Key Concepts Explained

🔐

JWT (JSON Web Token)

A compact, URL-safe token consisting of 3 parts: Header.Payload.Signature. Server signs it with a secret key. Client sends it on every request. Server verifies it without hitting the database — "stateless auth".

🔑

bcrypt Password Hashing

Converts plain passwords into an irreversible hash using a "salt" (random data). Even if DB is stolen, passwords can't be recovered. Uses 10 salt rounds by default in this project.

🔁

REST API

Architectural style using HTTP methods: GET (read), POST (create), PUT (update), DELETE (remove). Each endpoint is a URL representing a resource. Stateless — no session memory on server.

🌐

CORS

Cross-Origin Resource Sharing. Browser security blocks JS from calling APIs on different domains. Our backend explicitly allows localhost:5173 so React can talk to Express.

⚛️

React Context API

Solves "prop drilling" — passing data through many component levels. AuthContext stores user info globally. Any component can access it via useAuth() hook.

🍃

Mongoose ODM

Object Document Mapper — lets us define schemas (structure) for MongoDB collections. Provides methods like User.findOne(), User.create() instead of raw MongoDB queries.

Viva Preparation — Part 1

Expected Viva Questions

Easy

Q1. What is MERN stack? Which technologies are used in this project?

MERN = MongoDB, Express.js, React.js, Node.js. This project uses all four: MongoDB (database), Express + Node (backend), React (frontend). Additionally, JWT for auth, bcrypt for password hashing, and Axios for HTTP calls.

Easy

Q2. What is the purpose of the .env file? Why shouldn't it be committed to GitHub?

The .env file stores secret values like DATABASE_URL, JWT_SECRET, NEWS_API_KEY. If pushed to GitHub, anyone can read your secrets and misuse your API key or access your database. It is listed in .gitignore to prevent accidental commits.

Medium

Q3. What is JWT? How does it work in Newsly?

JWT (JSON Web Token) has 3 parts: Header (algorithm), Payload (user ID), Signature (HMAC with secret key). On login, the server signs a token and returns it. The frontend stores it in localStorage. On every protected API call, it's sent in the Authorization header. The backend verifies the signature without a DB lookup — making it "stateless".

Medium

Q4. Why is bcrypt used instead of storing passwords directly?

Plain text passwords are catastrophic if the DB is hacked. bcrypt applies a one-way hashing function with a random "salt" making two identical passwords produce different hashes. It uses "work factor" (10 rounds here) making brute-force attacks computationally expensive. The matchPassword method uses bcrypt.compare() to verify during login.

Medium

Q5. What is CORS and why is it needed in this project?

CORS (Cross-Origin Resource Sharing) is a browser security policy that blocks web pages from making requests to a different origin (domain:port). Our React frontend runs on port 5173 and talks to the Express backend on port 5000 — different origins. The cors() middleware on the backend explicitly allows this, so the browser permits the requests.

Viva Preparation — Part 2

More Viva Questions

Medium

Q6. What is the difference between authentication and authorization?

Authentication = verifying WHO you are (login with email/password). Authorization = verifying WHAT you're allowed to do (only logged-in users can access /api/saved). In Newsly, authMiddleware.js handles both: it first verifies the JWT (authentication), then attaches the user to req.user so the next handler can check permissions (authorization).

Medium

Q7. What is the MVC pattern? How is it applied in the backend?

MVC = Model-View-Controller. Model = MongoDB schemas (User.js, SavedArticle.js) — data layer. Controller = business logic (authController, newsController, savedController) — what to do with data. View = the response sent (JSON). Routes connect URLs to controllers. This separation makes code organized and maintainable.

Hard

Q8. What is the difference between SQL and NoSQL? Why did you choose MongoDB?

SQL (MySQL/PostgreSQL) uses tables with fixed schemas, strict relationships, ACID transactions. NoSQL (MongoDB) uses flexible JSON-like documents, no fixed schema, horizontally scalable. MongoDB was chosen because news article structures can vary (different fields), it integrates naturally with JavaScript/Node.js via JSON, and Mongoose makes schema validation optional but available.

Hard

Q9. What does the unique compound index on {user, url} in SavedArticle do?

MongoDB creates an index that enforces uniqueness on the combination of user ID and article URL. This means the same user cannot save the same article URL twice. If they try, MongoDB throws a duplicate key error (code 11000). The backend can catch this and return a "Already saved" response instead of a crash.

Hard

Q10. What is the React Context API and why was it used instead of passing props?

Context API provides a way to share state globally without passing it through every component level ("prop drilling"). AuthContext stores the logged-in user and login/logout functions. Any component in the tree — Header, PrivateRoute, SavedPage — can call useAuth() to access them directly. Alternative would be Redux, but Context is simpler for this project size.
